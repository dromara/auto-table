# AutoTable 数据库专属功能

每种数据库都有独特的特性和注解。本文覆盖各数据库的专属配置和最佳实践。

## MySQL

### 专属注解

| 注解 | 级别 | 说明 |
|------|------|------|
| `@MysqlEngine("InnoDB")` | 类 | 存储引擎 |
| `@MysqlCharset(charset, collate)` | 类 | 表字符集 |
| `@MysqlColumnCharset(value, collate)` | 字段 | 列字符集 |
| `@MysqlColumnUnsigned` | 字段 | 无符号数值 |
| `@MysqlColumnZerofill` | 字段 | 零填充数值 |
| `@MysqlFullTextIndex(parser)` | 字段 | 单列全文索引 |
| `@MysqlTableFullTextIndex(fields, parser)` | 类 | 多列全文索引 |

### 完整示例

```java
@Data
@AutoTable(comment = "文章表")
@MysqlEngine("InnoDB")
@MysqlCharset(charset = "utf8mb4", collate = "utf8mb4_general_ci")
public class Article {

    @PrimaryKey(autoIncrement = true)
    private Long id;

    @AutoColumn(comment = "标题", length = 200, notNull = true)
    private String title;

    @AutoColumn(comment = "内容")
    @ColumnType(MysqlTypeConstant.LONGTEXT)
    @MysqlFullTextIndex(parser = "ngram", comment = "内容全文索引")
    private String content;

    @AutoColumn(comment = "浏览量")
    @MysqlColumnUnsigned
    @ColumnDefault("0")
    private Integer viewCount;
}

// 多列全文索引
@AutoTable
@MysqlTableFullTextIndex(
    name = "ft_title_content",
    fields = {"title", "content"},
    parser = "ngram"
)
public class Article2 { ... }
```

### 配置项

```yaml
auto-table:
  mysql:
    table-default-charset: utf8mb4
    table-default-collation: utf8mb4_general_ci
    column-default-charset: utf8mb4
    column-default-collation: utf8mb4_general_ci
    alter-table-separate-drop: false  # 云数据库可能需要 true
    admin-user: root          # 自动建库用
    admin-password: xxx
```

### ENUM / SET 类型

```java
@ColumnType(value = "enum", values = {"male", "female", "other"})
private String gender;

@ColumnType(value = "set", values = {"read", "write", "admin"})
private String permissions;
```

### 类型常量速查（`MysqlTypeConstant`）

整数：`TINYINT` `SMALLINT` `MEDIUMINT` `INT` `BIGINT`
小数：`FLOAT` `DOUBLE` `DECIMAL`
字符串：`CHAR` `VARCHAR` `TEXT` `TINYTEXT` `MEDIUMTEXT` `LONGTEXT`
枚举：`ENUM` `SET`
日期：`YEAR` `TIME` `DATE` `DATETIME` `TIMESTAMP`
二进制：`BIT` `BINARY` `VARBINARY` `BLOB` `TINYBLOB` `MEDIUMBLOB` `LONGBLOB`
特殊：`JSON`

## PostgreSQL

### 专属配置

```yaml
auto-table:
  pgsql:
    pk-auto-increment-type: byDefault  # always 或 byDefault
    admin-user: postgres
    admin-password: xxx
```

### 自增主键

PostgreSQL 使用 `GENERATED BY DEFAULT AS IDENTITY`（byDefault）或 `GENERATED ALWAYS AS IDENTITY`（always）。

### 索引方法

PostgreSQL 支持额外索引方法：

```java
@Index(method = "gin")  // GIN 索引，适合 JSONB
private String metadata;

@TableIndex(
    name = "idx_gist_location",
    fields = {"location"},
    method = "gist"      // GiST 索引，适合地理数据
)
```

### 类型常量速查（`PgsqlTypeConstant`）

整数：`INT2` `INT4` `INT8`
小数：`FLOAT4` `FLOAT8` `MONEY` `NUMERIC`
字符串：`CHAR` `VARCHAR` `TEXT`
日期：`TIME` `TIMETZ` `DATE` `TIMESTAMP` `TIMESTAMPTZ`
二进制：`BYTEA`
布尔：`BOOL`
位串：`BIT` `VARBIT`
JSON：`JSON` `JSONB`
其他：`XML` `INTERVAL` `CIDR` `INET` `MACADDR` `TSQUERY` `TSVECTOR`

### JSONB 字段示例

```java
@AutoColumn(comment = "元数据")
@ColumnType(PgsqlTypeConstant.JSONB)
@Index(method = "gin")
private String metadata;
```

## Oracle

### 注意事项

- Oracle 表名/列名默认大写
- 无自增概念，使用 Sequence（AutoTable 自动处理 IDENTITY 列，Oracle 12c+）
- 字符串用 `VARCHAR2`，大文本用 `CLOB`

### 类型常量速查（`OracleTypeConstant`）

字符：`CHAR` `VARCHAR2` `NCHAR` `NVARCHAR2`
数值：`NUMBER` `BINARY_FLOAT` `BINARY_DOUBLE`
日期：`DATE` `TIMESTAMP` `TIMESTAMP_WITH_TIME_ZONE` `TIMESTAMP_WITH_LOCAL_TIME_ZONE`
大对象：`BLOB` `CLOB` `NCLOB` `BFILE`
行标识：`ROWID` `UROWID`
二进制：`RAW`

## SQLite

### 注意事项

- 类型系统较宽松（动态类型）
- 不支持 ALTER TABLE DROP COLUMN（旧版本）
- 不支持 schema 概念
- 自增主键使用 `AUTOINCREMENT`

### 类型常量

SQLite 没有独立的 TypeConstant 类，使用标准 SQL 类型名：`INTEGER`、`TEXT`、`REAL`、`BLOB`、`NUMERIC`。

## Doris

### 专属注解

Doris 是 OLAP 数据库，建表语法与 MySQL 差异大，需要配置数据模型、分区分桶。

```java
@Data
@AutoTable(comment = "事件日志")
@DorisTable(
    duplicate_key = {"id", "event_time"},
    distributed_by_hash = {"id"},
    distributed_buckets = 3,
    dynamic_partition = @DorisDynamicPartition(
        enable = true,
        time_unit = DorisTimeUnit.day,
        end = "3",
        start = "-7",
        prefix = "p",
        buckets = "3"
    )
)
public class EventLog {

    @PrimaryKey(autoIncrement = true)
    private Long id;

    @AutoColumn(comment = "事件时间", notNull = true)
    private LocalDateTime eventTime;

    @AutoColumn(comment = "事件类型")
    private String eventType;

    @AutoColumn(comment = "金额")
    @DorisColumn(aggregateFun = AggregateFun.sum)
    private BigDecimal amount;
}
```

### 数据模型

Doris 三种数据模型（三选一）：

| 模型 | 注解属性 | 适用场景 |
|------|---------|---------|
| 明细模型 | `duplicate_key` | 日志、原始数据 |
| 主键模型 | `unique_key` | 需要去重的数据 |
| 聚合模型 | `aggregate_key` | 指标汇总 |

### 索引类型（`DorisIndexType`）

| 类型 | 说明 |
|------|------|
| `inverted` | 倒排索引（默认） |
| `ngram_bf` | NGram BloomFilter 索引 |
| `bitmap` | 位图索引 |

### 配置项

```yaml
auto-table:
  doris:
    rollup-prefix: "auto_rlp_"
    rollup-auto-name-max-length: 100
    update-limit-table-data-length: 2147483648  # 2GB
    update-backup-old-table: false
```

### 大表更新限制

Doris 的 ALTER TABLE 是重建表操作，数据量大时非常慢。框架默认对超过 `update-limit-table-data-length`（默认 1GB）的表跳过 update，需手动处理。

### 类型常量速查（`DorisTypeConstant`）

整数：`TINYINT` `SMALLINT` `INT` `BIGINT` `LARGEINT`

日期：`DATE` `DATETIME`

聚合：`HLL` `BITMAP` `QUANTILE_STATE` `AGG_STATE`

## 多数据库适配（Dialect）

同一实体适配多种数据库：

```java
@AutoTable
public class Document {

    @PrimaryKey(autoIncrement = true)
    private Long id;

    @AutoColumns({
        @AutoColumn(type = "longtext", dialect = "MySQL"),
        @AutoColumn(type = "text", dialect = "PostgreSQL"),
        @AutoColumn(type = "clob", dialect = "Oracle"),
        @AutoColumn(type = "clob", dialect = "DM")
    })
    private String content;

    @AutoColumns({
        @AutoColumn(length = 100, dialect = "MySQL"),
        @AutoColumn(length = 200, dialect = "PostgreSQL")
    })
    private String title;
}
```

### 方言常量（`DatabaseDialect`）

| 常量 | 值 |
|------|-----|
| `MySQL` | `"MySQL"` |
| `MariaDB` | `"MariaDB"` |
| `PostgreSQL` | `"PostgreSQL"` |
| `Oracle` | `"Oracle"` |
| `DM` | `"DM"` |
| `KingBase` | `"KingBase"` |
| `H2` | `"H2"` |
| `SQLite` | `"SQLite"` |
| `Doris` | `"Doris"` |

匹配规则：未指定 dialect 的 `@AutoColumn` 作为默认值，框架根据当前 JDBC 连接自动选择匹配的配置。
